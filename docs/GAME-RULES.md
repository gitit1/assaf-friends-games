# 🎮 חוקי בניית משחקים

הכללים שכל משחק באפליקציה מקיים. **משחק חדש חייב לעמוד בהם.** (מבט-על: [CHECKLIST.md](CHECKLIST.md).)

## עקרונות-על (תמיד)
- **בלי שעון, בלי לחץ.**
- **בלי כישלון ובלי עונש** — דחיפה עדינה בלבד (`playNudge`), בלי צליל מפחיד. תשובה לא-נכונה פשוט לא מתקדמת.
- **תמונה + קול** (אפשר לכבות בהגדרות). ילד שאינו קורא חייב להבין את המשחק **בלי טקסט**.
- **מטרות מגע גדולות · RTL · מובייל-פירסט.**
- **חגיגה בסיום** (`Confetti` + קול). **בלי באנר "כל הכבוד" כתוב שקופץ** — הוא מזיז את הלייאאוט.

## רמות קושי — **מצטברות**
- 4 רמות: **קל / בינוני / קשה / אלוף**. הרמה ההתחלתית לפי `difficulty` בהגדרות (`levelForTier`).
- **כל רמה מגישה את הפריטים שלה + של כל הרמות שמתחתיה:**

| רמה | מאגר |
|---|---|
| קל | רק שלו |
| בינוני | קל + שלו |
| קשה | קל + בינוני + שלו |
| אלוף | **הכול** (כל הרמות) |

- לכן **אלוף הוא תמיד המאגר הגדול ביותר**. בבונים-מילה: אלוף ≥ 60 מילים.
- מימוש לדוגמה (`SpellWord.tsx`):
  ```ts
  const POOLS = LEVELS.map((_, i) => LEVELS.slice(0, i + 1).flat())
  // POOLS[0]=קל · POOLS[1]=קל+בינוני · POOLS[2]=...+קשה · POOLS[3]=הכול (אלוף)
  ```

## RTL ואנגלית
- ה-UI **עברי-ראשון** וממורכז ב-RTL; לבדוק שגם **LTR** נכון (project skill: `rtl-mirror`).
- תוכן **אנגלי** (מילים/אותיות) — `dir="ltr"` על השורה שמציגה אותו (כדי שייקרא שמאל-לימין).

## קול
- קול **מוקלט** (Narakeet) עם נפילה ל-TTS — דרך `playClip` / `src/voice.ts`.
- **אנגלית:** `speakEn()` ב-`src/speech.ts` כופה קול אנגלי של הדפדפן, ללא תלות בשפת ה-UI.
