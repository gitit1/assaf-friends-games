# 🎚️ התאמת רמה להורים — תכנית עבודה

## מטרה
לאפשר להורה לכוונן את הרמה לילד שלו, כך שהאפליקציה תתאים לכל גיל/שלב — לא רק לאסף.
**ברירת המחדל נשארת "רמת אסף"** (מתקדם: עד 100, כל הפעולות, אלוף), אבל ההורה יכול לשנות.

## מה ההורה יוכל לבחור (בהגדרות) — ✅ הוחלט
1. **טווח מספרים** — 5 / 10 / 20 / 50 / 100 (חמישה כפתורים קבועים).
2. **פעולות חשבון** — הדלקה/כיבוי לכל פעולה ➕ ➖ ✖️ ➗ (הרמה לפי "רמת קושי" הכללית).
3. **רמת קושי** — (כבר קיים: קל / בינוני / קשה / אלוף).
4. **האם הטווח מגביל גם את "החברים שלי"** — מתג לבחירת ההורה (הרוסטר כולו, או רק המשחקים).
5. **פריסטים** — "גן" / "כיתה א׳" / "אסף" (מכוונים הכול יחד), ועדיין אפשר לכוונן ידנית אחרי.

## מודל הנתונים (`settings.ts`)
```ts
maxNumber: number                       // 5/10/20/50/100, ברירת מחדל 100
ops: { add; sub; mul; div }             // בוליאנים, ברירת מחדל הכול true
limitRoster: boolean                    // האם הטווח מגביל גם את "החברים שלי", ברירת מחדל false
difficulty: number                      // קיים (0–3), ברירת מחדל 3 (אלוף)
```
נשמר ב‑localStorage כמו שאר ההגדרות; ברירות המחדל = רמת אסף ⇒ **אפס שינוי התנהגות** עד שמשנים.

## פריסטים
| פריסט | טווח | פעולות | קושי |
|---|---|---|---|
| 🧸 גן | 10 | ➕ ➖ | קל |
| 🎒 כיתה א׳ | 20 | ➕ ➖ | בינוני |
| 📘 כיתה ב׳–ג׳ | 50 | ➕ ➖ ✖️ | קשה |
| ⭐ אסף (ברירת מחדל) | 100 | ➕ ➖ ✖️ ➗ | אלוף |

בחירת פריסט מכוונת את כל ההגדרות; אפשר לכוונן ידנית אחרי (הפריסט הופך ל"מותאם אישית").

## עזר מרכזי חדש (`src/level.ts`)
מקור-אמת אחד שכל המשחקים קוראים ממנו, במקום תקרות קשיחות מפוזרות:
- `friendCap()` → `maxNumber` — כמה חברים קיימים/מוגרלים.
- `randFriend()` → אינדקס חבר אקראי בטווח `0..friendCap-1`.
- `numberMax()` → התקרה הכללית למספרים במשחק.
- `enabledOps()` → הפעולות הפעילות (לבוררים).
- `opRange(op)` → טווח אופרנדים לכל פעולה לפי `maxNumber` + `difficulty` (כך שהתוצאה ≤ `maxNumber`).

## מה צריך להשתנות (מקובץ)
**א. תקרת החברים (roster)** — כל `randInt(0, FRIENDS.length-1)` → `randFriend()`:
"החברים שלי" (להציג 1..maxNumber), ניווט עולם-חבר, וכל משחק שמגריל חבר.

**ב. משחקי מספרים (טווח)** — להחליף תקרות קשיחות ב‑`numberMax()`:
קפיצות · עשרות-ואחדות · גדול-או-קטן · חבר-חסר-ברצף · מספר-וכמות · מציירים-מספר · המספר-החסר.

**ג. משחקי חשבון (פעולות + אופרנדים)** — להציג רק פעולות פעילות, אופרנדים מ‑`opRange`:
בונים-מספר (בורר ➕➖✖️➗) · המספר-החסר · מחשבון (מצב אתגר) · אתגר-חשבון · מרוץ-מכוניות · מרוץ-חשבון.

**ד. הגדרות (UI)** — סקשן חדש "🎚️ רמה והתאמה" עם הבוחרים, + הסבר קצר להורה.

## שלבים (מדורג — ניתן לפרסם אחרי כל שלב)
1. **תשתית** — מודל נתונים + `level.ts` + ה‑UI בהגדרות (ברירות מחדל = אסף → אין שינוי). ✅ בטוח לפרסום.
2. **תקרת חברים** — roster לפי `maxNumber` (כולל "החברים שלי"). קל ובטוח.
3. **טווח במשחקי מספרים**.
4. **פעולות במשחקי חשבון**.
5. *(אופציונלי)* פריסטים.

## ברירת מחדל = אסף
`maxNumber=100`, כל הפעולות, `difficulty=3` — ההתנהגות הנוכחית נשמרת בדיוק.

## החלטות (סגור) ✅
1. טווח: **5 / 10 / 20 / 50 / 100** (כפתורים קבועים).
2. פעולות: **הדלקה/כיבוי לכל פעולה**; הרמה לפי "רמת קושי".
3. הגבלת "החברים שלי": **מתג לבחירת ההורה** (`limitRoster`).
4. **פריסטים: כן** (גן / כיתה א׳ / כיתה ב׳–ג׳ / אסף) + כוונון ידני.
