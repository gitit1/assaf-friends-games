# 🌍 מערכת שפות (i18n) — תכנון

מסמך-תכנון לפני יישום. המטרה: **מקור-אמת אחד** לכל מה שקשור לשפה — טקסט כתוב, טקסט מדובר, וקול מוקלט — עם **טוגל** עברית/אנגלית, מוכן להרחבה לעוד שפות.

> סטטוס: 🛠️ שלב 1 בוצע · עודכן 03/06/2026
>
> **החלטות:** קול = **Azure Neural** · שמות חברים = **תעתוק** (Lulu) · התחלה = **תשתית + מסך-דמה**.
>
> **בוצע (שלב 1):** `src/i18n/` (types/he/en/index, `t`/`say`/`useT`/`setLang`/`dirFor`) · שדה `lang` בהגדרות + **טוגל** + החלפת `dir` (RTL↔LTR) · `speech`/`voice` מודעי-שפה · קליפים עברו ל-`public/voice/he/` · **מסך-הבית מחובר ל-`t()`** (כולל הקטגוריות) · היצרן מודע-שפה + ספק **Azure**.
> **הבא:** לחלץ את שאר המחרוזות ל-`he.ts` (audit ניקוד) → לתרגם `en.ts` → להריץ Azure פר-שפה.

---

## 1. מטרות

- **עברית + אנגלית** עכשיו; ארכיטקטורה שתומכת בהוספת שפה ב-3 צעדים (מילון + קליפים + הוספה לרשימה).
- **מקור-אמת אחד** לכל מחרוזת — אין טקסט "קשיח" (hard-coded) בקומפוננטות.
- לכל מחרוזת שלוש שכבות: `text` (כתוב על המסך), `say` (מה שנאמר — עם **ניקוד** לעברית), ו-`clip` (קובץ קול מוקלט).
- **טוגל** בהגדרות (🇮🇱 עברית / 🇺🇸 English), נשמר ב-localStorage.
- **כיווניות** מתחלפת עם השפה: עברית = RTL, אנגלית = LTR.
- **קול טבעי** (לא רובוטי!) בשתי השפות.

---

## 2. עקרון-העל: כל מחרוזת היא `key`

במקום `<h1>סופרים</h1>` → `<h1>{t('game.count.title')}</h1>`.
במקום `speak('כל הכבוד')` → `say('fx.wellDone')`.
המספרים (1–50, דינמיים) מטופלים בנפרד ע"י `speakNumber(n)` (קליפ פר-שפה).

## 3. מבנה קבצים

```
src/i18n/

  he.ts           // המילון העברי  (text + say מנוקד)
  en.ts           // המילון האנגלי (text + say)

```

`src/settings.ts` → מוסיף שדה `lang: Lang` (כמו `voice`/`sound` הקיימים, עם אותו מנגנון `useSettings`).

## 4. מודל הנתונים (source of truth)

```ts
// types.ts
export type Lang = 'he' | 'en'
export type Entry = { text: string; say?: string } // say חסר ⇐ משתמשים ב-text

// he.ts  (say = עם ניקוד, רק היכן שצריך להקראה)
export const HE: Record<string, Entry> = {
  'app.title':        { text: 'עולם החברים' },
  'home.meet':        { text: 'החברים שלי' },
  'game.count.title': { text: 'סופרים' },
  'fx.wellDone':      { text: 'כל הכבוד!', say: 'כָּל הַכָּבוֹד!' },
  'world.hello':      { text: 'שלום!',     say: 'שָׁלוֹם!' },
  // …כל מחרוזת באפליקציה
}

// en.ts
export const EN: Record<string, Entry> = {
  'app.title':        { text: "Friends' World" },
  'home.meet':        { text: 'My Friends' },
  'game.count.title': { text: 'Count' },
  'fx.wellDone':      { text: 'Well done!' },
  'world.hello':      { text: 'Hello!' },
}
```

> **חוזה:** כל key חייב להופיע בשני המילונים. נוסיף בדיקת-בנייה קטנה (סקריפט) שמוודאת ש-`Object.keys(HE)` ≡ `Object.keys(EN)` — כך לא נשכח תרגום.

```ts
// index.ts (תמצית ה-API)
export function t(key: string, vars?: Record<string, string|number>): string  // טקסט כתוב
export function sayText(key: string): string   // הטקסט המדובר (say ?? text)
export function clipId(key: string): string    // מזהה קליפ = ה-key
export function useLang(): Lang                 // hook, מרנדר-מחדש בשינוי
export function setLang(l: Lang): void          // מעדכן setting + dir
export function dirFor(l: Lang): 'rtl' | 'ltr'
```

## 5. קול — מקור-אמת + טבעיות

- **שכבת הנגינה הקיימת (`src/voice.ts`) תורחב לפי שפה:** `playClip(id)` ינגן `public/voice/<lang>/<id>.mp3`, וייפול ל-TTS בשפה הנכונה אם הקליף חסר.
- **`speakNumber(n)`** → `voice/<lang>/num-<n>.mp3`.
- **משפט-UI מדובר** (`say('fx.wellDone')`) → `voice/<lang>/<key>.mp3`, נפילה ל-TTS עם `sayText`.

### הבעיה: חינם = רובוטי
ה-`gtx` (Google-translate) **מבטא נכון אבל מכני** — לא מתאים למוצר. לקול **טבעי** צריך TTS נוירוני, וזה דורש **מפתח חינמי** (חד-פעמי). לאחר מכן היצרן מוריד הכל **אוטומטית** (לא אחד-אחד).

### המלצה: **Microsoft Azure Neural TTS**
- עברית **טבעית** באמת: `he-IL-HilaNeural` (אישה) / `he-IL-AvriNeural` (גבר).
- אנגלית: עשרות קולות נוירוניים מצוינים (למשל `en-US-JennyNeural`).
- **שכבה חינמית: 500,000 תווים/חודש** — השימוש שלנו זעיר (~12K תווים לשתי השפות).
- מפתח אחד, ריצה אחת לכל שפה → היצרן `gen-voice.mjs` יקבל ספק `azure` + שפה.

חלופות: **Google Cloud** (אנגלית Neural2 מעולה; עברית Wavenet — סבירה, פחות טבעית מ-Azure) · **ElevenLabs** (אנגלית מעולה, עברית חלשה) · **הקלטה אנושית** (הכי טבעי ואישי — קול אמא! — אבל עבודה ידנית; אפשרי רק למספרים/משפטי-מפתח).

### ניקוד — מעבר טקסט-טקסט
לכל `say` עברי **נוודא ניקוד נכון, מחרוזת-מחרוזת** (זה מה שמתקן הגייה גם ב-TTS-נפילה וגם נשמע נכון בהקלטה). זהו שלב audit מסודר במילון `he.ts`.

## 6. כיווניות RTL ⇄ LTR (האתגר הגדול)

- בהחלפת שפה נעדכן `document.documentElement.lang` ו-`dir` (`rtl`/`ltr`).
- היום האפליקציה **RTL-first**, ויש מקומות שמכריחים `direction: ltr` (סטפר, dn-pad, בורר-עשרות, ביטויי-חשבון…) כדי לסדר מספרים/חצים. ב-LTR רובם הופכים ל-no-op נכון, אבל **צריך מעבר ידני מסך-מסך** לוודא.
- אייקונים/חצים שהם "קדימה/אחורה" — לבדוק שלא מתהפכים לא-נכון.

## 7. הטוגל

בהגדרות (⚙️) שורה חדשה: **🌍 שפה** → שני כפתורים `עברית` / `English`.
- בחירה → `setLang` → מעדכן setting (נשמר), מחליף `dir`, וכל הקומפוננטות מתרנדרות עם הטקסט החדש.
- ברירת-מחדל: עברית.

## 9. שלבי ביצוע (לפי הסדר)

1. **תשתית i18n** — `types/he/en/index`, שדה `lang` בהגדרות, טוגל, והחלפת `dir`. (בלי לתרגם הכל עדיין — רק המנגנון + מסך אחד לדוגמה.)
2. **חילוץ כל המחרוזות** מהקומפוננטות ל-`he.ts` (כולל audit ניקוד ל-`say`). הקומפוננטות עוברות ל-`t()`/`say()`.
3. **תרגום `en.ts`** (כל ה-keys) + תעתוק שמות חברים.
4. **יצרן קול נוירוני** (`gen-voice.mjs` + ספק `azure`, פר-שפה) → מייצר `voice/he/*` ו-`voice/en/*`.
5. **חיבור `voice.ts`** לבחור תיקיית-שפה; `speakNumber`/`say` לפי השפה.
6. **בדיקת RTL/LTR** בכל מסך + תיקונים.
7. בדיקת-בנייה: HE-keys ≡ EN-keys.

> כל שלב = קומיט נפרד על `dev`. שלב 1–2 הם הכבדים; 3–7 מצטברים.
